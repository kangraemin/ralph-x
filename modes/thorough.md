# Thorough Mode

Full pipeline with interview, design, review, and testing phases.

## Pipeline
1. **Interview** — Ask clarifying questions (up to 3 rounds until clear)
2. **Design** — Architecture decisions, API design, data models
3. **Develop** — Incremental implementation with checkpoints
4. **Review** — Self-review against requirements, check for edge cases
5. **Test** — Comprehensive testing, edge cases, error scenarios

## Behavior
- Stage 1: Pause and ask questions before writing any code
- Stage 2: Create design doc before implementation
- Stage 3: Implement in small increments, commit frequently
- Stage 4: Re-read requirements, verify completeness
- Stage 5: Write thorough tests, test edge cases
- Best for: complex features, greenfield projects, critical systems
